## **Сервис назначения ревьюеров для Pull Request’ов**

Внутри команды требуется единый микросервис, который автоматически назначает ревьюеров на Pull Request’ы (PR), а также позволяет управлять командами и участниками. Взаимодействие происходит исключительно через HTTP API.

## **Описание задачи**

Необходимо реализовать сервис, который назначает ревьюеров на PR из команды автора, позволяет выполнять переназначение ревьюверов и получать список PR’ов, назначенных конкретному пользователю, а также управлять командами и активностью пользователей. После merge PR изменение состава ревьюверов запрещено.

## Инструкция по запуску
Склонируйте репозиторий и перейдите в папку с проектом

```bash
git clone https://github.com/LyNuu/Service-for-assigning-reviewers-for-Pull-Requests
cd Service-for-assigning-reviewers-for-Pull-Requests
```

Запустите приложение с помощью docker команды
```bash
docker-compose up --build
```
Сервис будет доступен по адресу: `http://localhost:8080`

После запуска приложения, автоматически в БД будут тестовые данные, засчет миграции!

Далее перейдите в Postman и опробуйте работоспособность приложения:

```bash
# ✅ Ping сервиса
curl "http://localhost:8080/healthcheck"
```

```bash
# ✅ Успешное получение команды
curl "http://localhost:8080/team/get?team_name=avito"
```

```bash
# ✅ Успешное получение PR (пользователь u2 назначен ревьювером)
curl "http://localhost:8080/users/getReview?user_id=u2"
```

```bash
# ✅ Пользователь без PR (u4 не назначен ревьювером)
curl "http://localhost:8080/users/getReview?user_id=u4"
```

```bash
# ✅ Пользователь без PR (u4 не назначен ревьювером)
curl "http://localhost:8080/team/get?team_name=avito"
```
